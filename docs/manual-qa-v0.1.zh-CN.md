# Pig Studio v0.1 手工验收清单

## 1. 一分钟内打开项目并创建会话

1. 启动应用：`bun run dev`
2. 点击 **“选择项目文件夹”**，通过系统文件夹选择器添加一个本地项目目录
3. 在侧边栏中确认项目已出现在 `Project -> Session` 层级下，且项目名会自动从目录名推断
4. 点击新建会话
5. 在 1 分钟内确认主区域已显示会话头部、状态 badge 和输入区
6. 发送一个简单 prompt，确认时间线开始增长，且不会因为 UI 卡死而无法继续操作侧边栏与设置入口

## 2. 重启后恢复历史会话

1. 关闭应用
2. 再次启动应用
3. 确认左侧侧边栏中仍能看到上次项目与会话入口
4. 打开最近会话，确认主区域能恢复会话标题与基础时间线
5. 如果上次存在 `Running` 会话，确认应用启动后会自动尝试恢复 / 对账，而不是静默丢失该会话

## 3. 状态识别

分别验证以下状态能以正确 badge 呈现：

- `Running` -> `badge-info`
- `WaitingApproval` -> `badge-warning`
- `Blocked` -> `badge-secondary`
- `Failed` -> `badge-error`
- `Interrupted` -> `badge-neutral`

## 4. 审批与设置

1. 触发一个审批请求
2. 确认审批卡片显示在主区域中
3. 点击批准或拒绝，确认审批结果会写入时间线，而不是只在卡片上瞬时消失
4. 打开运行时面板，确认会自动显示：
   - 当前检测到的 Pi 二进制路径
   - 当前检测到的 Pi 配置目录
   - 检测来源（应用覆盖 / 环境变量 / PATH / 平台默认位置）
5. 使用“选择自定义二进制”或“选择自定义配置目录”验证覆盖能力
6. 点击“恢复自动检测”，确认界面会回到自动发现结果
7. 若运行时路径从不可用修复为可用，确认原先 `Blocked` 的会话可以继续恢复

## 5. Git / 非 Git 项目流程

1. 用一个 Git 仓库创建会话，确认可以进入正常会话流
2. 用一个非 Git 本地目录创建会话，确认不会因为缺少 Git 而被阻塞
3. 当 worktree 创建失败时，确认仍能回退到 direct 模式继续使用

## 6. 实时输出

1. 发送一个会产生多段输出的 prompt
2. 在运行尚未结束前观察时间线
3. 确认输出是逐步追加的，而不是在进程结束后一次性全部出现
4. 若运行持续数秒，确认状态 badge 在运行中保持为 `Running` 或 `WaitingApproval`

## 7. 恢复失败后的重建入口

1. 先创建一个正在运行或待恢复的会话
2. 在重新打开应用前，让 `pi` / `pi-mono` 暂时不可用（例如把运行时路径改成无效值，或移除环境中的可执行路径）
3. 重启应用并打开该会话
4. 确认会话以只读历史方式展示，且状态明确降级为 `Interrupted` 或 `Blocked`
5. 确认主区域出现 **“基于当前上下文新建会话”** 按钮
6. 点击该按钮，确认会在同一项目下创建新的后续会话，旧会话历史仍可继续查看
